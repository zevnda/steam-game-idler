# Troubleshooting

This page covers common issues and their solutions when using Steam Game Idler.

<Callout type="info">
    Always check the [FAQ](/docs/faq) section first, as many common questions are answered there
</Callout>

## Preliminary Checks

<Steps>
  <Step>
    ### Check System Requirements
    - Windows 10/11 (64-bit)
    - Internet connection
    - Steam client installed and running (only required if you use [Legacy Sign In](/docs/get-started/how-to-sign-in#legacy-sign-in))
  </Step>

  <Step>
    ### Verify Your Sign-In
    - If using [Sign in with Steam](/docs/get-started/how-to-sign-in#sign-in-with-steam), make sure you completed any Steam Guard prompt fully
    - If using [Legacy Sign In](/docs/get-started/how-to-sign-in#legacy-sign-in), make sure the Steam client is running and you're signed in to at least one account
    - Your Steam profile is set to public (required for some features regardless of sign-in method)
  </Step>

  <Step>
    ### Check SGI Installation
    - SGI is installed in the default location
    - You're running the latest version
    - Windows Defender/antivirus isn't blocking SGI
  </Step>
</Steps>

## Installation Issues

<Accordions type="multiple">
  <Accordion title="SGI won't install" id="install-fails">
    - Make sure you downloaded the installer from the official [GitHub releases page](https://github.com/zevnda/steam-game-idler/releases) or [steamgameidler.com](https://steamgameidler.com). Files from other sources may be incomplete or modified
    - Right-click the installer and select **Run as administrator**
    - Temporarily disable your antivirus/Windows Defender during installation (see below), then re-enable it afterwards
    - Make sure you have enough free disk space
    - If the installer hangs or crashes, restart your PC and try again
  </Accordion>

  <Accordion title="Antivirus or Windows Defender is blocking SGI" id="antivirus-blocking">
    SGI's installer and its bundled helper process (`SteamUtility.exe`, used for idling, achievement management, and Sign in with Steam) are not code-signed with a paid certificate, so Windows SmartScreen and some antivirus programs may flag them as "unknown publisher" or quarantine them. This is a false positive.

    <Steps>
      <Step>
        ### If SmartScreen blocks the installer
        Click **More info**, then **Run anyway**
      </Step>

      <Step>
        ### If your antivirus quarantines or deletes a file
        Restore the file from quarantine, then add an exclusion for SGI's install folder (default: `C:\Program Files\Steam Game Idler\`) and for `%appdata%\com.zevnda.steam-game-idler\`
      </Step>

      <Step>
        ### If a feature won't start because SteamUtility.exe is removed
        Re-run the installer to restore the missing file, after adding the exclusion above
      </Step>
    </Steps>

    <Callout type="info">
      If you'd like to verify the source code yourself, SGI's source code is entirely public. See the [GitHub repository](https://github.com/zevnda/steam-game-idler)
    </Callout>
  </Accordion>

  <Accordion title="Missing .NET Framework / SteamUtility errors" id="dotnet-missing">
    SGI's idling, sign-in, and achievement features rely on a bundled helper process (`SteamUtility.exe`) which requires the **.NET Framework 4.8** runtime. Most Windows 10/11 installs already include this, but if idling, sign-in, or achievement actions fail immediately with no clear error:

    - Install the [.NET Framework 4.8 runtime](https://dotnet.microsoft.com/en-us/download/dotnet-framework/net48) from Microsoft
    - Restart your PC after installing, then try again
  </Accordion>

  <Accordion title="I need admin rights to install or update SGI" id="admin-rights">
    SGI installs per-machine by default (available to all Windows users on the PC), which requires administrator privileges. If you don't have admin rights:

    - Ask an administrator to run the installer once
    - Updates are installed silently in the background and also require the original install permissions. If updates consistently fail, reinstall with an administrator account
  </Accordion>
</Accordions>

## Startup Issues

<Accordions type="multiple">
  <Accordion title="SGI won't start/run at all" id="wont-start">
    <Steps>
      <Step>
        ### Check for a hidden window or tray icon
        SGI's main window is hidden until it finishes loading, and may also be set to start minimized. Look for the SGI icon in your system tray (bottom-right of the taskbar, click the **^** arrow to see hidden icons)
      </Step>

      <Step>
        ### Check for leftover processes
        Open **Task Manager** and look for `steam-game-idler.exe` and `SteamUtility.exe`. If either is running but nothing appears, end both processes and relaunch SGI
      </Step>

      <Step>
        ### Check the log file
        If SGI ran before, open today's log file from <MockButton type='view-log-file' /> in [Debug Settings](/docs/settings/debug) and check it for errors from the last session
      </Step>

      <Step>
        ### Reset window state
        A corrupted window-position file can prevent the window from appearing. Close SGI completely (including via Task Manager), then delete `%appdata%\com.zevnda.steam-game-idler\.window-state.json` (if present) and relaunch
      </Step>

      <Step>
        ### Reinstall
        If none of the above works, uninstall SGI, restart your PC, and reinstall the latest version
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="SGI is open in the system tray but I can't see the main window" id="tray-no-window">
    This usually means **Start minimized to tray** is enabled, or the window was closed to tray and is simply hidden.

    - **Left-click** the SGI icon in the system tray to show and focus the main window
    - If clicking the tray icon does nothing, right-click it and select **Show**
    - If the window still doesn't appear, it may be positioned off-screen (see below)
    - To stop SGI from starting minimized, go to <MockButton type='context-cog' content='Settings → General' /> and disable **Start minimized**
    - Can also happen if **WebView2** is missing or outdated. Download the "Evergreen Bootstrapper" via [Microsoft's WebView2 download page](https://developer.microsoft.com/en-us/microsoft-edge/webview2/) and relaunch SGI
  </Accordion>

  <Accordion title="The window appears off-screen, too small, or in the wrong position" id="window-offscreen">
    SGI remembers its window size and position between sessions. If you changed monitor setups (e.g. disconnected a second monitor) the window may open off-screen.

    - Try pressing **Win + Shift + Left/Right Arrow** while the window is focused (use Alt+Tab to focus it) to move it back to your main display
    - Right-click the SGI taskbar icon and choose **Move** (or **Maximize**) if available
    - As a last resort, fully close SGI, delete `%appdata%\com.zevnda.steam-game-idler\.window-state.json`, and relaunch. This resets the window to its default size and position
  </Accordion>

  <Accordion title="SGI starts but crashes or closes immediately" id="crash-on-start">
    - Check the most recent log file (in the `logs` folder next to where SGI stores its settings, see [Debug Settings](/docs/settings/debug)) for the last recorded entries before the crash
    - Make sure your Windows is up to date. Tauri-based apps depend on the WebView2 runtime, which ships with Windows 10/11 but can be repaired via [Microsoft's WebView2 download page](https://developer.microsoft.com/en-us/microsoft-edge/webview2/) if missing or corrupted
    - If your settings file is corrupted, SGI may fail to start. Open the settings folder via <MockButton type='open-settings-file' /> in [Debug Settings](/docs/settings/debug) (or `%appdata%\com.zevnda.steam-game-idler\cache\` for an installed build) and rename or move `settings.json` elsewhere, then relaunch. SGI will recreate it with defaults
    - Try reinstalling the latest version
  </Accordion>
</Accordions>

## Sign-In Issues

<Accordions type="multiple">
  <Accordion title="'Sign in with Steam' fails or gets stuck" id="agent-signin-stuck">
    - Make sure you have an active internet connection
    - Double check your username and password. If you're not sure, try signing in on [steamcommunity.com](https://steamcommunity.com/) directly first
    - If using the QR code, make sure you scanned it with the **Steam** mobile app's own QR scanner (top-right icon), not a generic QR app, and approve the prompt on your phone
    - An unscanned QR code expires after a short time and will automatically refresh itself, so there's no need to reopen the sign-in screen
    - Close the sign-in window and try again. It can occasionally get stuck on a stale attempt
  </Accordion>

  <Accordion title="No Steam accounts show up on the Legacy Sign In screen" id="no-accounts">
    Legacy Sign In reads your locally saved Steam logins to populate the account picker.

    - Make sure the Steam client is installed and that you've logged in to your account(s) through Steam at least once
    - Open Steam and sign in normally, then restart SGI
    - If you use **Steam Guard**, complete the login fully (don't cancel at the 2FA prompt) before switching to SGI
    - If you'd rather not deal with the local Steam client at all, use [Sign in with Steam](/docs/get-started/how-to-sign-in#sign-in-with-steam) instead
  </Accordion>

  <Accordion title="A 'Steam is closed' warning appears while using SGI" id="steam-closed-warning">
    This only affects **Legacy Sign In** accounts, which depend on the local Steam client staying open and signed in. If Steam closes, is restarted, or you sign out of it while SGI is running, any active idling/farming/unlocking for that account is stopped and this warning appears until Steam is running and signed in again.

    - Reopen the Steam client and make sure you're signed in to the same account
    - This doesn't affect accounts signed in with [Sign in with Steam](/docs/get-started/how-to-sign-in#sign-in-with-steam), since they don't need a local Steam client at all
  </Accordion>

  <Accordion title="Signed in, but my games list is empty or incomplete" id="empty-games-list">
    - Your Steam profile (or at least your **game details**) must be set to **Public** for SGI to read your library. Check this in your Steam profile's privacy settings
    - Try clicking <MockButton type='content' content='Refresh' /> on the <MockButton type='your-games' /> page
    - Try toggling **Settings → Debug → Clear Data** as a last resort if the cached games list is stuck (note this signs out every account and clears all local data)
  </Accordion>
</Accordions>

## Idling Issues

<Accordions type="multiple">
  <Accordion title="A game won't start idling" id="idle-wont-start">
    - If using **Legacy Sign In**, make sure the Steam client is running and you're signed in to the same account you're using in SGI
    - Make sure the game is actually owned on this account. SGI can't idle games you don't own
    - Check the log file (<MockButton type='view-log-file' /> in [Debug Settings](/docs/settings/debug)) for a "Failed to start idling" message
    - If you recently reinstalled SGI, make sure `SteamUtility.exe` wasn't removed by your antivirus (see [Antivirus or Windows Defender is blocking SGI](#antivirus-blocking))
    - Close and reopen SGI, then try idling the game again
  </Accordion>

  <Accordion title="A game shows as idling in SGI but Steam doesn't show it as 'Currently in game'" id="idle-not-reflected">
    - Give it a minute, since there's a short delay before Steam's status updates
    - Make sure Steam itself is online (not in **Offline Mode**), as idling relies on Steam's servers being reachable
    - If using **Legacy Sign In**, restarting Steam (not SGI) while a game is idling can desync the status. Stop and restart idling for that game
  </Accordion>

  <Accordion title="Idling stops by itself" id="idle-stops">
    - Make sure your PC isn't going to sleep. Adjust Windows power settings so it stays awake while idling
    - If using **Legacy Sign In**, Steam will stop "in-game" status if you launch the same game manually, or if Steam restarts/updates in the background
    - Check the log file for errors around the time idling stopped
  </Accordion>

  <Accordion title="Leftover SteamUtility.exe processes after closing SGI or a crash" id="leftover-processes">
    If SGI is force-closed (e.g. via Task Manager, or a crash) before it can clean up, helper processes can be left running.

    - Open **Task Manager**, find any `SteamUtility.exe` entries, and select **End Task** for each
    - This won't affect your Steam library or achievements. It only stops the helper process
    - To avoid this in future, close SGI normally (via the tray menu **Quit** option) rather than force-closing it
  </Accordion>
</Accordions>

## Card Farming & Inventory Manager Issues

<Accordions type="multiple">
  <Accordion title='"Not logged in" or authentication errors' id="card-farming-not-logged-in">
    Card Farming and Inventory Manager rely on [Steam Credentials](/docs/steam-credentials), Steam Community session cookies which are separate from your SGI sign-in and expire periodically.

    - Go to **Settings → Steam Credentials** and re-authenticate (see the [Steam Credentials](/docs/steam-credentials) guide)
    - If you're on a **Gamer** tier account signed in with Steam, this normally happens automatically. Check that your subscription is still active
    - If you use **Family View** or have a **Steam Guard Mobile Authenticator**, make sure you complete the full login (these accounts often need the additional `steamMachineAuth` cookie)
    - Session cookies typically last a few hours to a few weeks. If farming worked before and suddenly stops with login errors, this is the most common cause
  </Accordion>

  <Accordion title="Card Farming isn't detecting card drops" id="card-farming-no-drops">
    - Make sure the games you're farming actually have **remaining card drops** for your account. Check your Badges page on Steam
    - Make sure your Steam profile's **game details/inventory** are set to Public
    - Re-authenticate if your session has expired (see above)
  </Accordion>

  <Accordion title="Listing cards on the Marketplace fails" id="market-listing-fails">
    - Make sure your Steam account doesn't have a **trade or market restriction** (new accounts, accounts without a phone number, or VAC-banned accounts can't use the Market)
    - Make sure you have **Steam Guard Mobile Authenticator** enabled for at least 7 days. Valve requires this for market access
    - If you're listing many items quickly, Steam may rate-limit you. Wait a few minutes and try again
    - Re-authenticate if your session cookies have expired
  </Accordion>
</Accordions>

## Achievement Manager & Unlocker Issues

<Accordions type="multiple">
  <Accordion title="Achievements list is empty or won't load" id="achievements-empty">
    - Make sure the game actually has achievements (some games genuinely have none)
    - Try toggling the **refresh/refetch** option for that game to force SGI to re-fetch the achievement schema from Steam
    - If using **Legacy Sign In**, make sure the Steam client is running and online
  </Accordion>

  <Accordion title="Unlocking, locking, or toggling an achievement fails" id="achievement-toggle-fails">
    - If using **Legacy Sign In**, make sure the Steam client is running and you're signed in to the matching account
    - Make sure the game isn't currently being idled by another tool, or already running through Steam itself, which can conflict with `SteamUtility.exe`
    - Check the log file for a specific error
    - Some games use **Family Sharing** or third-party anti-cheat that blocks stat writes. These games generally can't be modified
  </Accordion>

  <Accordion title="Achievement Unlocker isn't progressing" id="achievement-unlocker-stuck">
    - Make sure SGI is open (it can be minimized to tray, but not fully closed) for scheduled unlocking to continue
    - Check your unlock order, delay, and schedule settings under [Settings → Achievement Unlocker](/docs/settings/achievement-unlocker)
    - If you imported custom unlock timings, make sure the achievement names match exactly. See [Import Timings](/docs/features/achievement-unlocker/import-timings)
    - Check the log file for errors during the scheduled run
  </Accordion>
</Accordions>

## Settings & Data Issues

<Accordions type="multiple">
  <Accordion title="My settings reset, or SGI behaves like a fresh install" id="settings-reset">
    SGI stores app-wide settings in one `settings.json`, plus a separate file per feature (Achievement Unlocker, Card Farming, Inventory Manager, Free Games) for each signed-in Steam account. If any of these files becomes corrupted or unreadable, SGI falls back to that file's defaults automatically rather than failing to start.

    - Click <MockButton type='open-settings-file' /> in [Debug Settings](/docs/settings/debug) to locate your settings folder
    - If you have a previously exported settings backup, re-import it (see [Export Settings](/docs/settings/debug#export-settings))
    - Switching accounts in SGI loads separate per-feature settings files for that account. This is expected, not a bug
  </Accordion>

  <Accordion title="How do I fully reset SGI?" id="full-reset">
    <Steps>
      <Step>
        ### Try Reset Settings first
        Click <MockButton type='reset-settings' /> in <MockButton type='context-cog' content='Settings → Debug' />. This restores defaults without touching your games list or cached data
      </Step>

      <Step>
        ### Use Clear Data for a full wipe
        Click <MockButton type='clear-data' /> in <MockButton type='context-cog' content='Settings → Debug' />. This removes all locally stored data and signs out every account
        <Callout type="warn">
          This is irreversible and will require you to sign in and reconfigure SGI again. Your Steam account and library are not affected.
        </Callout>
      </Step>

      <Step>
        ### Manual reset
        If SGI won't open at all, close it completely and delete the folder `%appdata%\com.zevnda.steam-game-idler\`, then relaunch
      </Step>
    </Steps>
  </Accordion>
</Accordions>

## Update Issues

<Accordions type="multiple">
  <Accordion title='"Check for updates" does nothing, or no updates are found' id="update-check-nothing">
    - Make sure you have an active internet connection
    - SGI checks for updates via GitHub. If GitHub is unreachable from your network (e.g. blocked by a firewall/VPN), the check will fail silently
    - You can always download the latest release manually from the [GitHub releases page](https://github.com/zevnda/steam-game-idler/releases)
    - **Portable** builds don't support automatic updates. Download and replace the executable manually
  </Accordion>

  <Accordion title="Update downloads but SGI doesn't restart, or restarts into the old version" id="update-stuck">
    - Manually close SGI completely (including from the system tray and Task Manager), then reopen it
    - If the version number still hasn't changed, uninstall and reinstall using the latest installer from GitHub
  </Accordion>
</Accordions>

## General Performance Issues

<Accordions type="multiple">
  <Accordion title="SGI is using high CPU or memory" id="high-cpu-memory">
    - If using **Legacy Sign In**, idling many games simultaneously spawns a separate helper process per game. This is normal, but idling a very large number of games at once will increase resource usage. Accounts signed in with Steam idle through one shared session instead, which uses noticeably less
    - Check **Task Manager** to confirm whether the usage comes from `steam-game-idler.exe`, `SteamUtility.exe` or `webview2.exe` processes
    - Update to the latest version, as performance fixes are released regularly. See the [Changelog](/changelog)
  </Accordion>

  <Accordion title="SGI feels slow or unresponsive" id="sgi-slow">
    - A large Steam library (thousands of games) can make initial loading slower. Give it a moment on first launch after sign-in
    - Running several accounts signed in with Steam at once (see [Multi-Account](/docs/get-started/multi-account)) uses more memory and network than a single account. Reduce the number of concurrently signed-in accounts if resources are tight
    - Check the log file ([Debug Settings](/docs/settings/debug)) for repeated errors that may indicate a stuck background task
  </Accordion>
</Accordions>

## Getting Help

If these troubleshooting steps don't solve your issue:

1. **Check the [FAQ](/docs/faq)** for additional solutions
2. **Search existing issues** on the [GitHub repository](https://github.com/zevnda/steam-game-idler/issues)
3. **Create a new issue** with:
   - Detailed description of the problem
   - Steps to reproduce
   - Your system information (Windows version, Steam version)
   - Which sign-in method you're using (Sign in with Steam or Legacy Sign In)
   - Relevant log file excerpts
   - Screenshots if applicable

<Callout type="info">
    When reporting issues, never share your Steam credentials or personal information
</Callout>
