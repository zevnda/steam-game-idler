# Custom Order & Unlock Delay

The `custom order` feature lets you set the exact order in which achievements are unlocked, and allows you to skip any that you don't want unlocked.

The `unlock delay` is how long SGI waits after unlocking one achievement before unlocking the next. For example, if you set a delay of 10 minutes on achievement one, SGI will unlock achievement one, wait 10 minutes, and then unlock achievement two.

Per-achievement unlock delays override the global value set in [<MockButton type='context-cog' content='Settings → Achievement Unlocker → Unlock Interval' />](/docs/settings/achievement-unlocker#unlock-interval). If no per-achievement unlock delay is set, the [Unlock Interval](/docs/settings/achievement-unlocker#unlock-interval) value is used.

You can customize the order, set delays, or both. Each step is optional.

<Steps>
  <Step>
    Follow steps 1 and 2 from the [Default Mode](/docs/features/achievement-unlocker#default-mode) section
  </Step>

  <Step>
    Switch to the <MockButton type='content' content='Queue' /> tab
  </Step>

  <Step>
    Click <MockButton type='unlock-order' /> below the game you want to customize
  </Step>

  <Step>
    Drag and drop achievements by grabbing the <MockButton type='grabber' /> to set their unlock order
  </Step>

  <Step>
    Click <MockButton type='checkbox' /> to deselect achievements and skip them from being unlocked
  </Step>

  <Step>
    Enter a value (in minutes) in **Delay before first unlock** to set how long SGI waits before unlocking the very first achievement
  </Step>

  <Step>
    Enter a delay (in minutes) in the input field of each achievement to set how long SGI waits after unlocking that achievement before unlocking the next one
  </Step>

  <Step>
    Click <MockButton type='save-alt' /> and then click <MockButton type='achievement-unlocker-action' />
  </Step>
</Steps>
