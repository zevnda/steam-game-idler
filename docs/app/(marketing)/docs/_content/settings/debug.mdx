# Debug

You'll find these tools under <MockButton type='context-cog' content='Settings → Debug' />. If something isn't working as expected, start here before reporting a bug.

## Logs

The Logs panel displays a live view of SGI's internal activity, including errors and events from features like Card Farming and Achievement Unlocker.

- Click <MockButton type='view-log-file' /> to open today's log file
- Click <MockButton type='clear-logs' /> to clear the log view

<Callout type="info">
  Logs are written to a new file every day, kept in a `logs` folder next to where SGI stores its
  settings and cache: `<install folder>/logs/` for the Windows portable version, or your OS's
  app-data location otherwise (`%appdata%\com.zevnda.steam-game-idler\` on Windows,
  `~/.local/share/com.zevnda.steam-game-idler/` on Linux). <MockButton type='view-log-file' />
  always opens the right one for you, so you don't need to remember the exact path.
</Callout>

If you're reporting a bug on GitHub, include the relevant log files to help diagnose the issue. See the [Troubleshooting](/docs/troubleshooting) guide for more information.

## Reset Settings

<Callout type="warn">
  This action is irreversible
</Callout>

Click <MockButton type='reset-settings' /> to reset all of SGI's settings back to default

This is useful if you've misconfigured something and want to start fresh without wiping everything.

## Clear Data

<Callout type="warn">
  This action is irreversible. You will be signed out of every account in SGI.
</Callout>

Click <MockButton type='clear-data' /> to sign every account out, stop any running automation, and clear all locally-stored app data on this device (settings, feature queues, logs, recent searches, inventory locks, etc).

Use this as a last resort if SGI is behaving unexpectedly and [Reset Settings](/docs/settings/debug#reset-settings) hasn't resolved the issue.

## Export Settings

Click <MockButton type='export-settings' /> to copy your current SGI settings to the clipboard. Use this to:

- Back up your configuration before making changes
- Transfer your settings to another machine
- Attach to a bug report when opening an issue on GitHub

## Open Settings File

Click <MockButton type='open-settings-file' /> to open the folder where SGI stores its settings file in File Explorer. Useful for manually inspecting the file, or pasting in an exported settings backup to restore a previous configuration.
